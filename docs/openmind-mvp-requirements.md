# OpenMind MVP 1.0 需求文档

## 1. 产品概要

OpenMind 是一款面向个人知识管理的 Web 端思维导图软件，可在主流桌面浏览器中使用。MVP 1.0 采用 Web 应用技术路线，以 Markdown 文件作为核心数据源，让用户既可以通过传统导图方式从根节点逐步扩展，也可以直接编写 Markdown 文档生成导图。

核心目标：

- 让用户用 Markdown 管理可迁移、可备份、可同步的思维导图数据。
- 提供图形化导图编辑体验，支持快速新增、编辑、删除节点。
- 支持节点批注：批注默认不直接显示，用户悬浮节点时以气泡展示。
- 支持用户自主配置 WebDAV，实现多设备同步。
- 支持导图图片导出，便于分享。

## 2. 核心功能需求

### 2.1 导图编辑

- 用户可以新建一张思维导图。
- 导图默认包含一个根节点。
- 用户可以从任意节点新增子节点、编辑节点标题、删除节点。
- 节点之间采用树状父子关系。
- MVP 节点仅包含两类核心内容：节点标题、节点批注。
- 导图视图需要支持基础缩放、拖动画布、节点聚焦。
- 删除节点时，需要同时删除其全部子节点，并在操作前给出确认。

### 2.2 Markdown 编辑模式

- 应用提供“导图模式”和“Markdown 模式”两种模式，用户可手动切换。
- Markdown 是核心数据源。
- Markdown 标题映射规则：
  - H1 对应根节点。
  - H2-H6 按标题层级生成子节点。
  - 某个标题下、下一个同级或更高级标题前的正文内容，作为该节点批注。
- 当用户在 Markdown 模式修改内容后，切回导图模式时应用重新解析并生成导图。
- 当用户在导图模式新增、编辑、删除节点后，Markdown 内容同步更新。
- MVP 不支持非树状依赖、自定义链接关系、节点内富文本渲染。

### 2.3 批注气泡

- 每个节点可以拥有批注。
- 批注来源于该节点对应 Markdown 标题下的正文。
- 导图中默认只显示节点标题，不直接显示批注正文。
- 用户将鼠标悬浮在节点上时，若该节点存在批注，则展示气泡。
- 气泡展示 Markdown 正文的基础文本渲染结果。
- 鼠标移出节点后，气泡自动隐藏。
- 移动端或触控设备可用点击节点的方式触发批注展示。

### 2.4 文件能力

- 支持新建导图、导入 `.md` 文件、导出 `.md` 文件。
- 用户可以在浏览器中编辑导图内容，并通过下载方式保存 Markdown 文件。
- 应用可使用浏览器本地存储保存最近编辑内容，避免刷新页面后立即丢失。
- 应用需要显示当前文件的保存状态：已保存、未保存、同步中、同步失败。
- 支持将当前导图导出为图片，MVP 至少支持 PNG。
- 图片导出应包含当前完整导图，而不是仅当前视口。

### 2.5 WebDAV 同步

- 用户可在设置中配置 WebDAV：
  - 服务器地址
  - 用户名
  - 密码或应用密码
  - 远端同步目录
- 用户可以测试 WebDAV 连接。
- 同步策略采用“按最后修改时间自动覆盖”：
  - 本地文件较新时上传覆盖远端。
  - 远端文件较新时下载覆盖本地。
- 为降低误覆盖风险，被覆盖版本必须自动保存到浏览器本地历史记录中，并支持导出备份文件。
- 同步失败时不删除本地文件，并展示失败原因。
- 浏览器直连 WebDAV 时要求 WebDAV 服务端允许跨域访问；若服务端不支持 CORS，MVP 需要提示用户该服务暂不可用。
- MVP 仅支持单个 WebDAV 账号，不支持多账号、团队共享、高级鉴权、自定义请求头。

## 3. 主要界面与交互

### 3.1 主界面

- 顶部工具栏包含：新建、导入 Markdown、导出 Markdown、导出图片、同步、模式切换、设置。
- 主区域在导图模式显示思维导图画布。
- 主区域在 Markdown 模式显示 Markdown 编辑器。
- 底部或角落显示当前文件状态与最近同步时间。

### 3.2 导图模式

- 节点以树状导图布局展示。
- 用户可以通过节点旁的新增按钮添加子节点。
- 双击节点标题进入编辑。
- 悬浮节点展示批注气泡。
- 右键节点展示操作菜单：新增子节点、编辑标题、编辑批注、删除节点。

### 3.3 Markdown 模式

- 使用纯文本 Markdown 编辑器。
- 编辑器需要保留用户输入的 Markdown 内容。
- 应用在切回导图模式时校验标题结构。
- 若 Markdown 无 H1，应用提示用户补充根节点标题，或自动将文件名作为 H1。

### 3.4 设置界面

- 提供 WebDAV 配置表单。
- 提供连接测试按钮。
- 提供同步目录说明。
- 提供浏览器本地历史备份列表，并支持下载备份文件。

## 4. 数据与接口约定

### 4.1 Markdown 文件结构

示例：

```markdown
# 根节点

根节点批注内容。

## 子节点 A

子节点 A 的批注。

### 子节点 A-1

更深层级节点的批注。

## 子节点 B
```

解析规则：

- 标题文本作为节点标题。
- 标题层级决定父子关系。
- 标题下正文作为节点批注。
- 空正文表示无批注。
- 若存在多个 H1，首个 H1 作为导图根节点，后续 H1 及其子层级继续解析为根节点下的导图节点，并提示用户存在多个 H1。

### 4.2 内部节点模型

MVP 内部至少需要表达：

- `id`：应用运行时生成的节点唯一标识。
- `title`：节点标题。
- `note`：节点批注。
- `level`：Markdown 标题层级。
- `children`：子节点列表。

`id` 不写入 Markdown 文件，避免污染用户文档。

### 4.3 浏览器本地备份

- 当同步覆盖本地或远端内容前，应用保存被覆盖版本。
- 备份文件名包含原文件名、来源、本地时间戳。
- 备份内容存储在浏览器本地存储中，用户可在设置中查看并下载。

## 5. 非功能需求

- 跨平台：支持 Windows、macOS、Ubuntu 上的主流桌面浏览器。
- 本地优先：无 WebDAV 配置时，应用仍可通过浏览器本地存储、Markdown 导入和 Markdown 导出完整使用。
- 数据可迁移：核心内容必须可通过普通 Markdown 文件读取。
- 稳定性：同步失败、解析失败、导出失败不能导致原文件丢失。
- 性能：MVP 应流畅支持至少 500 个节点的导图浏览与编辑。
- 安全：WebDAV 密码或应用密码默认不持久化保存；若用户选择记住凭据，必须避免明文存储，并明确提示浏览器本地保存风险。
- 可用性：除 WebDAV 同步外，核心编辑、导入、导出能力不依赖网络。

## 6. 验收标准

- 用户可以在浏览器中新建导图，在导图模式新增多级节点，并导出为 Markdown。
- 用户可以导入已有 Markdown 文件，并正确生成树状导图。
- 用户可以在 Markdown 模式编辑标题和正文，切回导图模式后结构与批注正确更新。
- 用户悬浮有批注的节点时，可以看到气泡；无批注节点不展示空气泡。
- 用户可以配置 WebDAV、测试连接、手动触发同步。
- 当远端文件更新时间晚于本地文件时，本地内容被覆盖，旧本地版本进入浏览器本地历史备份。
- 当浏览器当前内容更新时间晚于远端文件时，当前内容上传覆盖远端，旧远端版本进入浏览器本地历史备份。
- 用户可以将完整导图导出为 PNG 图片。
- 应用可在 Windows、macOS、Ubuntu 的主流桌面浏览器上完成基本启动、导入、导出、编辑、同步流程。

## 7. MVP 不包含

- 多人实时协作。
- 账号系统。
- 多 WebDAV 账号。
- 云服务托管。
- 非树状节点依赖关系。
- 节点颜色、图标、标签、状态等高级属性。
- PDF、HTML、SVG 等完整导出套件。
- 自动三方文本合并。
- 插件系统。
- 移动端 App。

## 8. 测试计划

- Markdown 解析测试：覆盖 H1-H6、空批注、多段批注、缺失 H1、多个 H1。
- 导图编辑测试：新增、编辑、删除节点后 Markdown 输出正确。
- 批注测试：有批注展示气泡，无批注不展示。
- 文件测试：新建、导入 Markdown、导出 Markdown、导出 PNG。
- WebDAV 测试：连接成功、连接失败、CORS 不可用提示、上传、下载、按时间覆盖、备份生成。
- 跨平台测试：Windows、macOS、Ubuntu 的主流桌面浏览器上完成核心用户路径。
- 异常测试：网络中断、权限不足、Markdown 解析失败、导出失败。

## 9. 已确认假设

- MVP 1.0 面向个人知识管理，不做团队协作。
- 应用采用 Web 端技术路线，MVP 不做桌面客户端。
- Markdown 是核心数据源。
- 导图与 Markdown 采用模式切换，而不是默认左右双栏。
- 节点范围限定为标题和批注。
- WebDAV 同步采用按最后修改时间自动覆盖。
- 被覆盖版本必须保留浏览器本地历史备份。
- MVP 支持 Markdown 文件和 PNG 图片导出。
